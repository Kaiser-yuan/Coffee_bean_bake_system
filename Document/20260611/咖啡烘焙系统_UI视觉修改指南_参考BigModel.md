# 咖啡烘焙系统 UI 视觉修改指南

> 参考方向：智谱 BigModel 开放平台  
> 编写日期：2026-06-11  
> 适用范围：咖啡烘焙系统 Vue 前端

## 1. 修改目标

当前系统功能结构已经较完整，本轮修改不调整业务流程，主要解决以下视觉问题：

- 页面像传统后台模板，缺少统一的产品气质。
- 内容虽然完整，但主次不够清晰。
- 表格、筛选、指标和图表之间缺少稳定的视觉层级。
- 页面可能使用了过多边框、标签、颜色或独立卡片。
- 核心的曲线分析区域没有获得足够的视觉中心地位。

参考 BigModel 的 UI 风格后，本系统建议采用：

```text
轻量科技感
+ 明亮低对比背景
+ 高质量白色工作区
+ 蓝色主操作
+ 克制的青绿色业务强调
+ 清晰的信息层级
+ 少量精致动效
```

这里借鉴的是视觉秩序和组件语言，不照搬 BigModel 的品牌、文案或营销页面布局。

明确借鉴边界：

```text
参考：
颜色克制、字体层级、留白秩序、组件质感、清晰状态

不参考：
营销大标题、宣传型首屏、大面积装饰渐变、过度留白、品牌视觉照搬
```

本系统是高频使用的数据分析工具，页面信息密度和操作效率优先于官网式视觉表现。

## 2. 整体视觉方向

### 2.1 关键词

```text
轻
净
专业
精确
安静
现代
数据优先
有呼吸感
```

### 2.2 页面基本结构

推荐使用三层结构：

```text
应用背景层
-> 页面内容层
-> 关键任务工作区
```

- 应用背景使用极浅的冷灰蓝，不使用纯白铺满整个页面。
- 页面主体使用白色或近白色工作区。
- 当前最重要的任务，例如曲线图、三级表格或评价结果，占据最大空间。
- 次要数据使用弱化文本、摘要条或浅色面板展示。

### 2.3 不建议继续使用的设计

- 大面积咖啡棕色、深橙色或复古色。
- 每一块内容都做成独立悬浮卡片。
- 强阴影、发光效果、玻璃拟态。
- 页面中大量彩色状态标签。
- 大面积渐变背景。
- 使用多个相互竞争的主色。
- 所有标题都加粗、所有指标都突出。

## 3. 新视觉变量

### 3.1 基础颜色

```css
:root {
  --app-bg: #f5f7fb;
  --surface: #ffffff;
  --surface-subtle: #f8faff;
  --surface-selected: #eef4ff;

  --text-primary: #172033;
  --text-secondary: #5d6880;
  --text-tertiary: #8b95a7;

  --border-default: #e5e9f2;
  --border-strong: #d7ddea;

  --primary: #2f6bff;
  --primary-hover: #2559d9;
  --primary-subtle: #edf3ff;

  --business-accent: #168477;
  --business-accent-subtle: #eaf7f4;

  --success: #1f9d68;
  --warning: #d78a16;
  --danger: #d94b4b;
}
```

### 3.2 色彩使用规则

- 蓝色只用于主要操作、选中状态、链接和当前导航。
- 青绿色用于咖啡业务状态，例如库存充足、目标达成或风味相关强调。
- 橙色只用于提醒和待处理状态。
- 红色只用于错误、库存不足和破坏性操作。
- 普通信息尽量使用文字层级和背景层级表达，不轻易使用彩色标签。

### 3.3 曲线颜色

曲线颜色需要保持专业分析工具的高区分度，不完全跟随页面主色：

```text
豆温：#df5b45
环境温：#e5a029
ROR：#3478d4
火力：#8b5cc7
风门：#20a184
搅拌：#718096
```

曲线颜色应仅在图表、图例和关联指标中使用，不扩散到普通按钮和页面装饰。

## 4. 字体与排版

### 4.1 字体建议

```text
中文：Noto Sans SC / 思源黑体
英文与数字：IBM Plex Sans
代码与批次编号：IBM Plex Mono
```

### 4.2 字号层级

```text
页面标题：22px / 600
区块标题：16px / 600
表格与正文：14px / 400
辅助信息：12px / 400
核心指标数字：24px / 600
```

规则：

- 不使用过大的后台页面标题。
- 文字不使用负字间距。
- 数据字段、温度、重量和百分比启用等宽数字。
- 主标题与页面操作同一行，减少无效纵向空间。
- 区块标题使用正常文字层级，不给每个区块增加独立标题卡片。

## 5. 布局与间距

### 5.1 页面框架

```text
左侧导航：224px，折叠后 64px
顶部栏：56px
页面内容最大宽度：不强制限制，分析页充分利用屏幕
普通页面左右留白：24px
大屏左右留白：32px
```

### 5.2 间距系统

统一使用 `4px` 基础间距：

```text
4px：图标与文字微间距
8px：紧密组件内部间距
12px：表单控件和工具栏间距
16px：普通内容组间距
24px：页面区块间距
32px：主要区域间距
```

避免随意使用大量不同间距，导致页面看起来松散或凌乱。

### 5.3 容器规则

- 按钮、输入框和表格容器圆角统一使用 `6px`。
- 抽屉和弹窗圆角使用 `8px`。
- 状态标签圆角使用 `4px`。
- 表格、图表和主要工作区使用一层容器，不嵌套卡片。
- 使用 `1px` 浅边框表达层级。
- 默认不使用阴影；浮层、下拉菜单和弹窗可使用轻阴影。

## 6. 导航与应用壳层修改

### 6.1 左侧导航

参考 BigModel 控制台的轻量导航风格：

- 导航背景使用白色。
- 右侧使用一条浅边框与工作区分隔。
- 图标统一使用线性图标。
- 未选中菜单使用次要文字色。
- 当前菜单使用浅蓝背景、蓝色图标和主文字色。
- 不使用高饱和整块背景。
- 系统设置固定在导航底部。

推荐状态：

```text
默认：透明背景 + 次要文字
悬停：#f5f7fb 背景
选中：#edf3ff 背景 + #2f6bff 图标
```

### 6.2 顶部栏

顶部栏保持安静：

- 左侧显示页面标题和必要的面包屑。
- 第一版右侧只保留用户菜单。
- 暂不增加没有明确需求的全局搜索和通知入口。
- 搜索功能放在各业务页面内部，贴近实际数据范围。
- 不在顶部栏堆放多个主要操作。
- 页面主要操作放在页面标题栏右侧。

## 7. 通用组件修改

### 7.1 按钮

按钮只保留三种主要层级：

```text
Primary：蓝色实心，用于当前页面唯一主要操作
Secondary：白底细边框，用于普通操作
Text / Icon：无边框，用于低频和行内操作
```

规则：

- 一个页面首屏最多一个蓝色主按钮。
- “首屏最多一个主按钮”属于默认建议，不是强制规则；当页面存在两个同等重要且高频的动作时，可以同时展示，但必须通过文字和位置区分优先级。
- 删除、关闭问卷等操作使用红色文字按钮，避免默认高亮。
- 图标按钮必须有 tooltip。
- 按钮高度统一为 `32px` 或 `36px`。
- 避免每行表格都出现多个文字按钮，优先使用更多菜单。

### 7.2 输入框和筛选器

- 高度统一为 `36px`。
- 使用浅边框，聚焦时显示蓝色描边。
- 筛选器集中放在一条工具栏内。
- 低频筛选放入“更多筛选”抽屉或弹层。
- 已生效筛选使用可关闭的轻量筛选标签。
- 搜索框应占据工具栏最大的宽度。

### 7.3 表格

表格是系统核心组件，需要显著提升质量：

- 表头使用极浅冷灰蓝背景。
- 表头文字使用次要文字色和中等字重。
- 行高建议 `48px` 至 `52px`。
- 行之间使用细分隔线，不使用每个单元格边框。
- 悬停行使用浅蓝背景。
- 展开行使用轻微层级背景，不增加厚重卡片。
- 数值右对齐，名称和描述左对齐。
- 状态优先使用色点加文字，而不是实心胶囊标签。
- 操作列固定在右侧。

### 7.4 状态标签

仅在确有必要时使用状态标签：

```text
已完成：绿色小点 + 已完成
待烘焙：橙色小点 + 待烘焙
解析失败：红色小点 + 解析失败
评价中：蓝色小点 + 评价中
```

标签背景保持很浅，不使用高饱和实心颜色。

状态不能只依赖颜色表达，必须同时提供文字或图标，兼顾色觉障碍和低对比屏幕。

### 7.5 弹窗与抽屉

- 生豆录入使用右侧抽屉，宽度约 `480px` 至 `560px`。
- 简短确认使用弹窗。
- 表单按信息组分区，不使用多个嵌套卡片。
- 底部操作栏固定，避免长表单滚动后找不到保存按钮。

## 8. 各页面具体修改指南

### 8.1 系统总览

目标：从“很多统计卡片”改成“年度烘焙工作台”。

建议结构：

```text
页面标题 + 年份切换

核心指标横向摘要条

烘焙频率主图             豆种 / 产区分布

待烘豆列表               最近烘豆列表
```

具体修改：

- 核心指标使用同一横向白色容器，指标之间用细分隔线，不使用四张悬浮卡片。
- 烘焙频率图作为页面视觉重点，占据更宽空间。
- 豆种和产区分布在宽屏下可并排展示，在较窄屏幕下切换展示，不为了视觉简化而隐藏重要数据。
- 待烘和最近烘豆使用紧凑表格。
- 待烘完成勾选使用简单 checkbox，不增加操作按钮。
- 图表点击后进入已带筛选条件的烘焙分析页。

### 8.2 生豆管理

目标：让三级关系清晰，而不是像复杂目录树。

建议结构：

```text
页面标题                         录入生豆
搜索框 + 常用筛选 + 更多筛选

三级树状表格
```

具体修改：

- 生豆档案一级行使用白色背景和较强名称层级。
- 采购批次二级行使用浅蓝灰背景。
- 烘焙批次三级行使用更紧凑的普通行。
- 每一级使用缩进、线性图标和背景变化表达关系，不使用厚边框。
- 总库存和剩余库存使用数字与细进度条展示。
- 库存状态不使用固定 `5000g` 阈值。优先根据剩余库存占采购总量的比例判断，或允许用户在系统设置中配置阈值。
- `+ 待烘计划`只出现在采购批次悬停或更多菜单中，减少视觉噪声。
- 录入抽屉顶部先搜索已有豆款，再决定显示基础字段还是采购字段。

### 8.3 烘焙批次列表

目标：突出筛选、状态判断和多锅选择。

具体修改：

- 页面顶部使用一条紧凑筛选工具栏。
- 选中批次后，在表格上方显示浮动批量操作栏。
- 对比按钮仅在选择 2 至 4 个有效批次时出现，并显示已选数量。
- 状态拆为曲线、评价、复盘三个小状态点，避免一行出现多个大标签。
- 生豆名称为主信息，豆种、处理法和产区作为第二行弱化信息。
- 时间、投豆量、失重率等数字统一对齐。

### 8.4 曲线分析页面

目标：让曲线成为绝对主角，其他信息退到辅助层。

推荐结构：

```text
顶部：返回 + 批次名称 + 批次切换 / 加入对比 + 进入复盘

指标摘要条

大面积曲线工作区

事件时间轴

评价摘要 / 对比指标
```

具体修改：

- 曲线图不放入多层卡片，使用单一大工作区。
- 图例放在曲线工作区顶部，可直接点击显隐。
- 主指标默认全部显示，保留 M1 的使用习惯。
- 曲线控制使用图标按钮和下拉菜单，避免大量文字按钮。
- KPI 使用横向摘要条，不使用多张指标卡。
- 单锅模式以指标颜色区分曲线。
- 多锅模式默认以批次颜色区分，使用线型和图表区域区分指标。
- 多锅模式提供“按批次对比”和“按指标对比”两种查看方式。
- 当前悬停时间点的数据使用固定浮动检查器展示，方便横向比较。
- 评价摘要放在图表下方，不抢占曲线空间。

#### 曲线编码规则

单锅模式：

```text
豆温：红色实线
环境温：黄色细实线
ROR：蓝色虚线
火力：紫色阶梯线
风门：绿色阶梯线
搅拌：灰色阶梯线
```

多锅模式默认：

```text
颜色：区分不同烘焙批次
线型：区分豆温、环境温、ROR
图表区域与阶梯线：区分火力、风门、搅拌
```

多锅“按指标对比”模式：

```text
先选择一个指标
-> 不同颜色区分不同烘焙批次
-> 减少同时显示的曲线数量
```

多锅模式不得仅依赖颜色同时表达批次和指标，否则用户无法可靠识别曲线含义。

### 8.5 批次复盘

目标：像一份持续更新的分析记录，而不是多个表单卡片。

推荐结构：

```text
左侧主内容：
个人初始复盘
评价反馈摘要
综合复盘

右侧固定摘要栏：
批次指标
下一锅建议
提醒 1 / 2 / 3
```

具体修改：

- 使用清晰的时间线或步骤标题表达复盘阶段。
- 文本编辑区域保持大面积、安静和易写。
- 评价摘要使用轻量图表与关键结论，不使用大量彩色进度条。
- 下一次烘焙提醒使用编号列表，不使用三张卡片。
- 宽屏下右侧摘要栏在滚动时保持可见。
- 普通笔记本或窄屏下，摘要栏移动到主内容顶部或底部，不压缩文本编辑区域。
- “创建下一次待烘计划”作为页面唯一主按钮。

### 8.6 评价管理

目标：让用户快速判断哪些问卷还在收集、反馈质量如何。

具体修改：

- 顶部显示轻量统计摘要：进行中、已结束、总评价数、平均综合喜好。
- 问卷列表以状态和提交数量为主要扫描字段。
- 复制链接使用图标按钮。
- 关闭问卷放入更多菜单或危险操作区。
- 评价详情使用雷达图或横向评分条时保持颜色克制。
- 酸感和苦感必须标注为强度，避免被误读为质量。

### 8.7 公开评价问卷

目标：手机端清爽、快速、不产生评价暗示。

具体修改：

- 使用窄内容列，桌面最大宽度约 `640px`。
- 顶部只展示生豆名称、烘焙日期和简短说明。
- 每个评分维度使用稳定高度，避免点击时布局跳动。
- 酸感和苦感明确显示“强度”。
- 风味标签使用轻量可选标签，不使用大量高饱和色。
- 提交按钮固定在表单底部，不使用悬浮遮挡内容。
- 提交成功页保持简洁，不展示他人评价。
- 公开问卷以填写效率和可读性为第一目标，不为了统一后台科技感而增加装饰或复杂交互。

### 8.8 系统设置

目标：低频但清晰，不占据系统主要视觉资源。

具体修改：

- 使用左侧二级设置导航或顶部紧凑标签页。
- 词表使用简单表格，不使用卡片网格。
- 启停操作使用开关。
- 已使用词条展示引用次数。
- 文件和备份状态使用简洁状态行。

## 9. 动效与交互反馈

动效只用于表达状态变化，不作为装饰。

动效属于后续优化项。第一版优先保证表格、曲线、表单、响应式和状态反馈正确。

推荐：

- 页面进入使用轻微淡入，时长 `160ms` 至 `220ms`。
- 抽屉和弹窗使用标准滑入或淡入。
- 表格展开使用高度过渡。
- 图表筛选与显隐使用平滑更新。
- 保存成功使用短暂 Toast。
- 加载使用骨架屏或局部加载，不让整个页面反复闪烁。

避免：

- 大幅缩放。
- 弹跳动画。
- 长时间渐变动画。
- 每个元素依次出现的复杂动画。

## 10. 推荐修改顺序

### 第一阶段：建立统一视觉基础

1. 建立 CSS 设计变量。
2. 修改应用背景、左侧导航和顶部栏。
3. 统一字体、字号、间距、边框和圆角。
4. 统一按钮、输入框、状态和表格组件。
5. 明确单锅与多锅曲线编码规则。

### 第二阶段：修改高频核心页面

1. 烘焙批次列表。
2. 曲线分析页面。
3. 生豆管理三级表格。
4. 批次复盘页面。

### 第三阶段：修改聚合与外部页面

1. 系统总览。
2. 评价管理与详情。
3. 公开评价问卷。
4. 系统设置。

### 第四阶段：统一检查

1. 删除多余卡片和阴影。
2. 检查主按钮数量。
3. 检查状态颜色是否过多。
4. 检查表格数字是否对齐。
5. 检查曲线图是否占据主要空间。
6. 检查桌面和手机端是否存在遮挡或溢出。
7. 检查状态是否同时包含文字或图标，不只依赖颜色。
8. 检查普通笔记本宽度下复盘页右侧摘要栏是否压缩正文。

## 11. 验收清单

### 全局

- [ ] 页面整体为明亮冷灰蓝背景与白色工作区。
- [ ] 蓝色只承担主要操作和选中状态。
- [ ] 页面没有大面积咖啡棕色和装饰性渐变。
- [ ] 普通内容主要依靠边框、留白和文字层级区分。
- [ ] 页面首屏最多一个主要蓝色按钮。
- [ ] 若首屏存在多个高频主操作，已经通过位置和样式明确主次。
- [ ] 所有核心数字使用等宽数字并正确对齐。

### 组件

- [ ] 卡片圆角不超过 `8px`。
- [ ] 页面区块没有多层卡片嵌套。
- [ ] 表格没有完整单元格边框。
- [ ] 状态主要使用色点加文字。
- [ ] 状态不只依赖颜色表达。
- [ ] 图标按钮均有 tooltip。
- [ ] 筛选栏紧凑，低频筛选被收起。

### 核心页面

- [ ] 曲线分析页面中，曲线是最大且最显眼的区域。
- [ ] 单锅模式使用指标颜色，多锅模式使用批次颜色和指标线型，不存在颜色含义冲突。
- [ ] 多锅模式支持按批次对比和按指标对比。
- [ ] 生豆三级关系通过缩进和背景层级清晰表达。
- [ ] 烘焙批次列表可以快速扫描状态并选择多锅。
- [ ] 批次复盘能清晰区分个人复盘、评价摘要和综合复盘。
- [ ] 公开评价问卷在手机端能够顺畅填写。
- [ ] 库存状态使用比例或可配置阈值，不使用固定 `5000g` 判断所有采购批次。
- [ ] 普通笔记本和窄屏下，复盘摘要不会压缩主要文本区域。

## 12. 一句话设计判断标准

每次修改页面时，用下面的问题检查：

> 用户打开这个页面后，能否在三秒内看出当前最重要的数据和下一步可以执行的操作？

如果不能，应优先减少视觉噪声、强化主任务，而不是继续增加装饰。
