# 咖啡烘焙系统：前端 v2 修复说明

日期：2026-07-02  
分支：`master`  
修改范围：前端源码与本说明文档。

## 1. 修改背景

本次修改基于前端代码 review 后的 v1 / v2 修改指南，目标是先在代码层修复已确认的问题，不做本机前后端联调。

核心目标：

1. 修复烘焙分析列表 loading / error / table 条件渲染问题。
2. 修复新建待烘计划默认生豆和采购批次可能为空的问题。
3. 修复曲线对比缺少对齐事件时整页报错的问题。
4. 完善从曲线页发起评价问卷后的分发闭环。
5. 暂时隐藏未真实保存的烘焙度字段，避免假字段。
6. 增强问卷链接复制在服务器 HTTP 访问场景下的兼容性。
7. 增强待烘计划默认生豆选择，避免选到已归档或失效生豆。

## 2. 修改文件

### 2.1 `frontend/src/views/RoastingBatchList.vue`

修改内容：

1. 调整 loading / error / table 的条件渲染结构。
2. 将 `contextWarning` 放入正常内容区域，避免它破坏 `v-if / v-else-if / v-else` 链。
3. 新增 `ensureCreateFormDefaults()`，用于自动补齐新建待烘计划的默认生豆和采购批次。
4. `ensureCreateFormDefaults()` 会先检查当前生豆是否仍在 active 生豆列表里；如果不在，会自动切换到第一个 active 生豆。
5. 隐藏“烘焙度”字段，因为当前前后端尚未真正保存该字段。

对应修复：

- 烘焙列表在 loading / error 时显示旧表格的问题。
- 第一次打开“新建待烘计划”时生豆或采购批次为空的问题。
- 已归档 / 失效生豆仍可能被默认选中的边缘问题。
- 烘焙度字段“看起来可填，但实际不保存”的假闭环问题。

### 2.2 `frontend/src/api/curves.ts`

修改内容：

1. 曲线对比接口请求增加 `strict=false`。

对应修复：

- 多锅曲线对比时，如果某锅缺少转黄、一爆开始、出豆等对齐事件，后端可以降级返回 warning，而不是直接让前端整页报错。

### 2.3 `frontend/src/views/CurveAnalysis.vue`

修改内容：

1. 发起评价问卷后保存问卷 ID 和 `shareCode`。
2. 增加“复制问卷链接”按钮。
3. 增加“查看评价详情”按钮。
4. 新增 `writeClipboardText()`，在 `navigator.clipboard.writeText()` 失败时使用 textarea fallback。
5. 如果 fallback 仍失败，弹窗显示完整问卷链接，方便手动复制。

对应修复：

- 发起评价问卷后，还需要跳到评价管理页面找链接的问题。
- 服务器部署在普通 `http://IP` 时，浏览器可能拒绝 Clipboard API 的问题。

## 4. 后续联调建议

在真实开发机器或服务器上，建议按下面顺序验收：

1. 启动后端并执行数据库迁移。
2. 启动前端，确认 `.env.local` 中 `VITE_DEMO_MODE=false`。
3. 进入烘焙分析页，确认 loading / error / 空状态显示正常。
4. 点击“新建待烘计划”，确认默认生豆和采购批次自动填充。
5. 上传两条曲线并进入对比页，确认缺少对齐事件时显示 warning，而不是整页报错。
6. 在单锅曲线页发起评价问卷，点击“复制问卷链接”，确认链接可打开。
7. 使用普通 `http://服务器IP` 访问时，再次测试复制链接 fallback。

## 5. 当前结论

本次修改没有改变数据库结构，也没有改变后端接口。

修改重点是修复前端主流程中的闭环断点，让下面这条链路更顺：

```text
生豆采购 -> 建立烘焙批次 -> 上传曲线 -> 曲线对比 -> 发起评价问卷 -> 复制链接分发 -> 汇总评价
```
