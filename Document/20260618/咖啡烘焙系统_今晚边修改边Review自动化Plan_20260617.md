## 1. 今晚主线

今晚不直接进入多 CSV 页面开发，先把“真实可联调”的地基修稳。

主线顺序：

```text
P0 当前代码风险修复
→ P1 前端真实 API / mock 边界整理
→ P2 多 CSV 与历史补录的数据模型准备
→ 每轮小改后立即 review 和验收
```

## 2. 工作循环

每一轮都按同一套节奏推进：

```text
选一个小任务
→ 读相关代码
→ 修改最小闭环
→ 运行对应检查
→ review diff
→ 更新 TODO 状态
→ 记录验收结果
```

每轮推荐控制在 20-45 分钟内。一次只处理一个闭环，避免同时改迁移、认证、前端 mock 和新功能。

## 3. 角色分工

用户负责：

- 确认今晚优先级是否需要调整。
- 在需要业务判断时拍板，例如历史补录是否允许缺采购批次。
- 根据 Codex 汇报选择继续、暂停或切换任务。

Codex 负责：

- 读取代码并判断最小修改范围。
- 直接修改代码、文档和测试。
- 每轮修改后做 review。
- 执行可运行的本地检查。
- 把完成情况同步到 TODO 和 Acceptance 文件。

## 4. 阶段计划

### 阶段 A：当前 P0 修复

目标：清除会阻塞真实联调或部署演进的问题。

范围：

- 重写 Alembic initial migration，移除 `Base.metadata.create_all/drop_all`。
- 给内部接口补认证，尤其是生豆树和匹配接口。
- 检查公开问卷接口是否仍保持匿名可访问。

完成信号：

- 迁移文件可读、可审查、可演进。
- 未登录访问内部接口返回 401。
- 公开问卷读取和提交不被误伤。

### 阶段 B：前端联调边界修复

目标：让用户清楚知道当前页面到底是 mock 还是后端真实 API。

范围：

- 登录页不再预填密码。
- `.env.example` 明确 demo 和 real 两套配置。
- 检查 `VITE_DEMO_MODE=false` 时登录守卫是否生效。
- 梳理页面直接 import mock 的位置，先迁移最影响联调判断的页面。

完成信号：

- 登录页没有默认密码。
- 真实 API 模式下未登录会进入登录页。
- 页面直接 import mock 的范围被明确记录，已处理项从 TODO 中勾掉。

### 阶段 C：库存与批量导入建模准备

目标：先把多 CSV 和历史补录需要的数据字段放进模型和迁移，为后端 preview/commit 铺路。

范围：

- 给 `roasting_batches` 增加：

```text
entry_mode
inventory_effective
roasted_at_source
bulk_import_group_id
source_note
```

- 新增：

```text
bulk_import_jobs
bulk_import_items
```

- 修改库存计算，使其只统计 `status='completed'` 且 `inventory_effective=true` 的烘焙批次。

完成信号：

- 模型、schema、迁移一致。
- 库存计算规则可审查。
- 历史补录默认不影响库存的规则有数据结构承载。

### 阶段 D：后端 bulk preview / commit 设计落地

目标：先做后端能力，不急着做页面。

范围：

- 设计并实现批量 CSV 预览接口。
- 支持文件解析、时间推断、warnings、重复检测、预计库存。
- 设计并实现提交接口。
- commit 时创建 completed roasting batch、curve file、active curve，并按规则扣库存。

完成信号：

- 预览不创建烘焙批次、不扣库存。
- commit 事务清晰，能防重复提交。
- 一个采购批次下多个 CSV 可以生成多个 completed 批次。

## 5. 每轮 Review 模板

每轮修改后，用下面结构做简短 review：

```text
本轮目标：
改动文件：
关键逻辑：
风险点：
已运行检查：
验收结果：
下一轮建议：
```

## 6. 暂停条件

出现以下情况时先暂停，不继续扩大修改面：

- 数据库模型和现有代码关系不清楚，继续改会误伤库存或曲线闭环。
- Alembic 当前状态无法判断是空库、已有库还是混合状态。
- 前端真实 API 缺少后端接口，导致页面迁移无法形成闭环。
- 本地依赖缺失导致关键检查无法运行，需要先决定是否安装或跳过。

## 7. 今晚不做的事

- 不做 RAG。
- 不做对话 Bot。
- 不做服务器部署。
- 不先做完整多 CSV 前端页面。
- 不把历史补录和日常多 CSV 导入混成一个入口。
