## 1. 系统概述

### 1.1 产品定位

面向咖啡烘焙师的数据分析工具，核心价值是记录生豆与采购信息，并通过烘焙曲线、杯测评价和批次复盘，持续分析不同烘焙操作对风味的影响。

### 1.2 两条核心数据流

```
生豆数据流：
生豆档案 → 采购批次 → 库存与原料状态

烘焙曲线分析流：
采购批次 → 烘焙批次 → M1 CSV 解析 → 曲线展示与对比
→ 个人初始复盘 → 问卷评价 → 综合复盘 → 下一锅建议
```

### 1.3 数据对象关系

```
生豆档案 1 ←→ N 采购批次
采购批次 1 ←→ N 烘焙批次
烘焙批次 1 ←→ 1  烘焙曲线
烘焙批次 1 ←→ N  评价问卷
评价问卷 1 ←→ N  杯测评价记录
烘焙批次 1 ←→ 1  批次复盘
```

### 1.4 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | Vue 3 + TypeScript |
| 路由 | Vue Router 4 |
| 状态管理 | Pinia |
| 图表 | ECharts |
| 构建 | Vite |
| Mock 数据 | 内建 mock 层 |

### 1.5 视觉风格

采用「现代实验室仪表盘」风格：扁平、明亮、紧凑、数据优先。页面背景 `#F5F7F8`，内容区域白色，主操作色 `#167D6A`。曲线指标采用固定颜色体系：豆温红、环境温黄、ROR 蓝、火力紫、风门绿、搅拌灰。

## 2. 应用壳层

### 2.1 左侧导航

固定在屏幕左侧，宽度 220px，支持折叠至 56px。

| 导航项 | 路由 | 功能说明 |
|--------|------|----------|
| 📊 系统总览 | `/dashboard` | 年度概览、待烘清单、最近记录 |

所有页面均包含三种状态：

| 状态 | 组件 | 表现 |

| 错误 | `ErrorState` | 警告图标 + 错误信息 + 重试按钮 |

## 3. 系统总览

**路由**：`/dashboard`

### 3.1 年份切换

页面右上角提供年份切换器，默认显示当前年份。切换后所有模块同步刷新。

### 3.2 年度核心指标（KPI）

四张指标卡片，展示：

| 指标 | 说明 |
|------|------|
| 已完成锅数 | 当年标记为"已完成"的烘焙批次数量 |
| 已烘豆款数 | 当年涉及的生豆种类数（去重） |
| 总投豆重量 | 当年所有已完成批次的投豆重量累加 |
| 平均失重率 | 当年所有已完成批次的失重率平均值 |

### 3.3 月度烘焙频率

以柱状图展示 1-12 月的烘焙锅数分布，柱色为 `#167D6A`。

### 3.4 豆种分布

环形饼图展示当年度烘焙豆种的比例分布。

### 3.5 产区分布

横向柱状图展示当年度烘焙产区的数量分布，柱色为 `#E06B45`。

### 3.6 待烘豆列表（仅展示）

按生豆名称分组的两级可展开列表：

- **一级**：生豆名称 + 豆种/处理法/产区 + 待烘计划数量
- **二级**（展开后）：烘焙目标、计划日期、投豆量、采购批次号、**勾选完成**

操作：
- 点击一级行展开/收起该豆款下的待烘计划
- 勾选二级行的 checkbox → 该批次状态变为"已完成"，数据同步刷新

> 注意：此处不提供录入功能。待烘计划的创建入口在「生豆管理」和「烘焙分析」页面。

### 3.7 最近烘豆列表

表格展示最近 8 条已完成的烘焙记录：烘焙时间、生豆名称、豆种、投豆/出豆量、失重率、状态。

点击任意行 → 跳转到该批次的曲线分析页。

## 4. 生豆管理

**路由**：`/green-beans`

### 4.1 搜索与筛选

顶部筛选栏包含：

| 控件 | 说明 |
|------|------|
| 搜索框 | 按生豆名称或豆种模糊匹配 |
| 豆种下拉 | 筛选豆种 |
| 处理法下拉 | 筛选水洗/日晒/蜜处理/厌氧发酵/半水洗 |
| 产区下拉 | 筛选产区 |

筛选只影响第一级生豆档案行，展开后不受影响。

### 4.2 三级树状表格

```
生豆档案 📁  ← 第 1 级，点击展开
├── 采购批次 📦  ← 第 2 级，点击展开
│   ├── 烘焙批次 🔥  ← 第 3 级，点击进入曲线
│   └── 烘焙批次 🔥
├── 采购批次 📦  ← [+ 待烘计划] 按钮在此
│   └── 烘焙批次 🔥
```

**第 1 级 — 生豆档案**

包含：生豆名称（加粗）、豆种、处理法、产区、海拔、**剩余库存**（带颜色圆点指示）
- 绿色 = 库存充足、黄色 = 库存偏低、红色 = 无库存

**第 2 级 — 采购批次**

包含：采购批次号、采购日期、总量、含水率、供应商、剩余库存（只读）
- 每行右侧有 `+ 待烘计划` 按钮

**第 3 级 — 烘焙批次**

包含：烘焙编号、日期、投豆/出豆量、失重率、状态标签、烘焙目标
- 点击行 → 跳转到曲线分析页

### 4.3 库存指示规则

| 库存范围 | 颜色 | 状态 |
|----------|------|------|
| > 5000g | 绿色 | 充足 |
| 1 - 5000g | 黄色 | 偏低 |
| ≤ 0 | 红色 | 耗尽 |

> 库存为后端计算值，前端只读展示，不可直接修改。

### 4.4 录入生豆（右侧抽屉）

点击 `+ 录入生豆` 打开右侧抽屉：

**搜索已有生豆**
在"生豆名称"输入框中输入 → 下方显示匹配的已有档案：

- **选择已有生豆** → 进入"新增采购批次"模式，只需填写采购信息
- **输入新名称（无匹配）** → 进入"录入新豆款"模式，需填写完整豆款信息 + 采购信息

**录入表单字段**

生豆信息（新模式）：
- 生豆名称、豆种、处理法、产区、农场、海拔

采购信息（始终显示）：
- 采购日期、采购总量（g）、含水率（%）、供应商、批次号

**保存**：点击保存后同时创建生豆档案和首个采购批次（或追加采购批次），列表即时刷新。

### 4.5 创建待烘计划（弹窗）

在采购批次行点击 `+ 待烘计划` → 弹出创建窗口：

- 自动带入：生豆名称、采购批次号、剩余库存（只读参考）
- 需填写：计划烘焙日期、投豆量（g）、烘焙目标
- 创建后该计划出现在三级列表中，同时反映到总览页的待烘列表

### 4.6 库存调整记录

点击采购批次 → 查看该批次的库存调整记录列表（日期、调整量、原因）。

## 5. 烘焙分析

**路由**：`/roasting`

### 5.1 筛选栏

| 筛选条件 | 说明 |
|----------|------|
| 生豆名称 | 文本搜索 |
| 豆种 | 下拉 |
| 处理法 | 下拉 |
| 产区 | 下拉 |
| 状态 | 待烘 / 已完成 |
| 曲线 | 有曲线 / 无曲线 / 不限 |
| 重置 | 清空所有筛选条件 |

### 5.2 批次列表表格

| 列 | 说明 |
|----|------|
| 对比勾选框 | 仅"有曲线"的批次可勾选，无曲线的置灰 |
| 颜色点 + 烘焙时间 | 批次颜色标识 + 日期 |
| 生豆名称 | — |
| 豆种 / 处理法 / 产区 | 关联自生豆档案 |
| 投豆量 / 出豆量 | — |
| 失重率 | 百分比 |
| 总时长 | m:ss 格式 |
| 曲线状态 | 无曲线 / 已解析 / 解析失败 |
| 评价状态 | 未发起 / 进行中 / 已关闭 |
| 复盘状态 | 未复盘 / 草稿 / 已完成 |
| 批次状态 | 待烘 / 已完成 |

### 5.3 操作

| 操作 | 触发方式 | 说明 |
|------|----------|------|
| 进入单锅曲线 | 点击表格行 | 跳转 `/curve/:batchId` |
| 多锅对比 | 勾选 2-4 个批次后点击按钮 | 跳转 `/curve/compare/:ids` |
| 新建待烘计划 | 点击 `+ 新建待烘计划` | 弹出创建窗口 |

**多锅对比规则**：
- 只能勾选 2-4 个批次
- 无曲线批次不可勾选
- 勾选满 2 个后顶部出现橙色对比按钮

### 5.4 新建待烘计划弹窗

字段：生豆下拉、采购批次下拉（按选中生豆过滤，显示剩余库存）、计划日期、投豆量、烘焙目标。

## 6. 曲线分析与多锅对比

**路由**：`/curve/:batchId`（单锅）、`/curve/compare/:ids`（多锅）

### 6.1 页面头部

- 左侧：返回按钮 + 页面标题 + 生豆名称/日期
- 右侧：批次颜色标识 + "进入复盘"按钮（仅单锅模式）

### 6.2 关键指标 KPI（仅单锅模式）

| 指标 | 显示格式 |
|------|----------|
| 总时长 | m:ss |
| 发展时间 | m:ss |
| 发展率 | xx.x% |
| 失重率 | xx.x% |
| 投豆量 | xxxg |
| 出豆量 | xxxg |

### 6.3 主图表

**图表布局**：
- 上层（40% 高度）：豆温、环境温、ROR 三条曲线，共用温度 y 轴（左）+ ROR y 轴（右）
- 下层（16% 高度）：火力、风门、搅拌三条阶梯线，共用百分比 y 轴

**单锅模式**：
- 默认显示全部 6 个指标通道
- 点击图例切换该指标的显隐
- 悬停数据点显示同一时刻的全部指标值
- 关键事件用虚线标记：入豆、回温点、转换点、一爆开始、一爆结束、下豆
- 事件标签在图表下方以标签条形式展示

**多锅对比模式**：
- 每个批次使用固定颜色（红、蓝、绿、紫）
- 支持按**指标**隐藏（如隐藏所有批次的 ROR）
- 支持按**批次**隐藏（如隐藏批次 2 的全部曲线）
- 同一批次在曲线、指标和评价区域使用相同颜色

### 6.4 曲线颜色体系

| 曲线 | 颜色 | 色值 |
|------|------|------|
| 豆温 | 红 | `#D6543D` |
| 环境温 | 黄 | `#D99A2B` |
| ROR | 蓝 | `#2677B8` |
| 火力 | 紫 | `#A34FA3` |
| 风门 | 绿 | `#278B72` |
| 搅拌 | 灰 | `#687477` |

### 6.5 事件时间轴

单锅模式下，图表下方以彩色左边框标签展示关键事件节点及其时间。

### 6.6 进入批次复盘

单锅模式顶部有"进入复盘"按钮 → 跳转 `/review/:batchId`。

## 7. 批次复盘

**路由**：`/review/:batchId`

### 7.1 批次概览

顶部展示 6 项基本数据：投豆量、出豆量、失重率、总时长、烘焙度、评价数。

### 7.2 个人初始复盘

- 自由文本区域，在外部反馈（评价）到达前填写
- 一旦保存即记录时间戳
- 可编辑，但个人初始复盘内容不被综合复盘覆盖

### 7.3 评价反馈摘要

- 如有评价数据，系统自动生成摘要（含提交人数和主要结论）
- 如无评价，显示空状态，提供「发起公开评价问卷」按钮
- 如有进行中的问卷，显示状态和提交人数，提供「关闭问卷」按钮

### 7.4 综合复盘

- 结合曲线数据和评价反馈的综合分析区域
- 保存时记录时间戳，支持后续更新

### 7.5 下一锅调整建议

- 自由文本区域，记录下一锅的操作调整方向

### 7.6 下一次烘焙提醒

- 最多三条，带优先级编号 #1 #2 #3
- 支持动态增减（上限 3 条）
- 要求简短、明确、可执行
- 提供「从此创建下一次待烘计划」按钮 → 自动带入提醒内容和烘焙目标到新的待烘计划

## 8. 评价管理

**路由**：`/evaluations`

### 8.1 问卷汇总列表

表格字段：关联批次号、生豆名称、烘焙时间、问卷状态（进行中/已关闭/已过期）、提交数、创建时间、截止时间。

> 注意：此页面**不能创建问卷**。问卷只能从具体的烘焙批次发起。

### 8.2 操作

| 操作 | 说明 |
|------|------|
| 点击行 | 进入评价详情页 |
| 关闭 | 关闭进行中的问卷 |
| 复制链接 | 复制公开问卷 URL 到剪贴板 |

### 8.3 评价详情

**路由**：`/evaluations/:questionnaireId`

**信息栏**：问卷状态、提交人数、分享链接（一键复制）、关闭按钮

**各维度平均分**：7 个维度的柱状进度条（干香、湿香、酸感、甜感、苦感、回味、综合喜好），每个维度使用独立颜色

**高频风味描述**：按出现频次排序的风味标签云，字号/透明度反映频次

**评价明细表**：评价者、类型（烘焙师/同事/顾客）、7 个维度分数、养豆天数、风味描述。综合喜好分数字体加粗突出。

## 9. 公开评价问卷

**路由**：`/eval/:shareCode`（无需登录）

### 9.1 设计特点

- 独立于后台管理界面，评价者通过链接或二维码访问
- 移动端友好（表单区域自适应手机屏幕）
- 不展示曲线、豆商描述或他人评价

### 9.2 评分维度（五分制）

| 维度 | 类型 | 描述 |
|------|------|------|
| 干香 | 质量 | 干咖啡粉的香气 |
| 湿香 | 质量 | 注水后的湿香 |
| 酸感 | 强度 | 酸味的强度 |
| 甜感 | 质量 | 甜味的感受 |
| 苦感 | 强度 | 苦味的强度 |
| 回味 | 质量 | 吞咽后的余味 |
| 综合喜好 | 偏好 | 整体的喜欢程度 |

评分方式：五星点击（☆ → ★），支持半星视觉（实际为 1-5 整数分）。

### 9.3 风味描述

从预设风味标签中多选（花香、柑橘、莓果、巧克力、坚果、焦糖、发酵、酒香等），选中后高亮。

### 9.4 自由备注

可选的自由文本区。

### 9.5 评价者信息

- 称呼（可选）
- 身份（顾客 / 同事 / 烘焙师）

### 9.6 提交

- 所有评分维度完成后方可提交（未完成时提交按钮禁用并显示提示）
- 提交后显示感谢页面
- 提交后自动关联正确的烘焙批次

### 9.7 过期与关闭

- 问卷有截止时间
- 过期或关闭后访问显示错误状态
- 烘焙师可在评价管理或复盘页关闭问卷

## 10. 系统设置

**路由**：`/settings`，4 个标签页：

### 10.1 标准词表

管理三类固定词表：风味描述、缺陷描述、烘焙度。

- 每个词条显示：值 + 已使用次数
- 已使用的词条 → 「停用」按钮（仅停用，不可删除）
- 未使用的词条 → 可自由切换启用/停用状态
- 停用后不再出现在录入控件的选项中

### 10.2 烘焙机与 CSV 配置（占位）

- 烘焙机配置：等待 M1 面板信息确认
- CSV 解析配置：等待 M1 CSV 字段说明确认

### 10.3 用户管理（占位）

第一版为单用户模式，多用户功能待后续版本。

### 10.4 文件与备份

三个状态项展示：
- 本地 CSV 文件存储状态
- 阿里云 OSS 自动每日备份状态
- PostgreSQL 数据库备份状态

## 11. 前后端协作契约

### 11.1 接口设计原则

- 核心接口围绕数据对象和业务动作
- 聚合接口围绕完整页面和分析场景
- 前端不直接承担库存、失重率、养豆天数等业务计算（由后端计算后返回）

### 11.2 数据对象聚合

```
GET /api/green-beans              → 生豆列表
GET /api/green-beans/:id/tree     → 三级树数据
POST /api/green-beans             → 创建生豆档案
POST /api/purchase-batches        → 创建采购批次
GET /api/roasting-batches          → 批次列表（支持筛排分页）
POST /api/roasting-batches         → 创建待烘计划
PUT /api/roasting-batches/:id/complete → 标记完成
POST /api/roasting-batches/:id/upload  → CSV 上传
GET /api/curves/:batchId           → 单锅曲线
GET /api/curves/compare?ids=...    → 多锅曲线
POST /api/questionnaires           → 发起问卷
GET /api/questionnaires            → 问卷列表
GET /api/questionnaires/:id/evaluations → 评价明细
PUT /api/questionnaires/:id/close  → 关闭问卷
POST /api/evaluations              → 提交评价（公开）
GET /api/reviews/:batchId          → 复盘数据

GET /api/terms                     → 词表

GET /api/dashboard?year=2026       → 总览聚合数据
```

### 11.3 Mock 数据说明

当前所有数据均为前端 Mock：
- 数据文件：`src/mock/index.ts`
- 支持模拟：成功、空数据、失败三种状态
- 当前延迟：300-500ms，无失败率
- 可通过 `setMockConfig({ delay, failRate })` 调整

### 11.4 页面状态覆盖

每个页面均实现：

| 状态 | 表现 |
|------|------|
| 加载中 | LoadingState 居中 |
| 正常有数据 | 表格/图表/表单正常展示 |
| 空数据 | EmptyState + 引导提示 |
| 加载失败 | ErrorState + 重试按钮 |
| 按钮禁用 | 不可操作时置灰 + tooltip 说明 |
