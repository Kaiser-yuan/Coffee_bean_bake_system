## 3. 核心闭环已跑通

```
录入生豆和采购批次                          ← 生豆管理页
  → 创建待烘计划                            ← 生豆管理 / 烘焙分析 / 复盘页
    → 标记完成 / 查看单锅曲线                 ← 总览页 / 烘焙分析页
      → 选择 2-4 锅曲线对比                  ← 烘焙分析页
        → 从批次发起评价问卷                  ← 复盘页
          → 查看评价汇总                      ← 评价管理 / 评价详情页
            → 完成批次复盘与下一次提醒         ← 复盘页
```

## 6. 下一步建议

### 6.1 短期内

1. **获取 M1 CSV 样本** → 对曲线页面进行最终定稿和 CSV 解析配置
2. **真实后端对接** → 按 `功能说明书.md` 第 11 章的接口设计启动后端开发
3. **Mock 替换** → 逐步将 `mock/index.ts` 中的函数替换为真实 API 调用

### 6.2 对接顺序建议

```
1. 生豆管理接口（GET/POST 生豆和采购批次）
2. 烘焙批次接口（列表、创建、完成、曲线）
3. 总览聚合接口（年度仪表盘数据）
4. 曲线接口（单锅、多锅对比）
5. 问卷与评价接口
6. 复盘接口
7. 设置接口
```

### 6.3 响应式与可用性
