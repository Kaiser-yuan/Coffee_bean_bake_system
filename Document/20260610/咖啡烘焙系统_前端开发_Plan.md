# 咖啡烘焙数据分析系统：前端开发 Plan

## 1. 开发目标

先使用 Mock 数据实现一套可运行、可点击、可验证交互的前端原型。原型确认后，再根据页面真实数据需求设计并接入后端接口。

第一版优先跑通以下闭环：

```text
录入生豆和采购批次
-> 创建或完成烘焙批次
-> 查看单锅曲线
-> 选择多锅曲线对比
-> 从批次发起评价问卷
-> 查看评价汇总
-> 完成批次复盘与下一次提醒
```

## 2. 实施原则

- 先完成信息架构和关键交互，再完善视觉细节。
- 所有页面先接 Mock 数据，不等待后端接口。
- Mock 数据结构作为前后端共享契约的初稿。
- 库存、失重率、养豆天数和统计值由后端负责计算，前端只展示。
- 页面必须包含加载、空数据、错误和禁用状态。
- 桌面端优先，同时保证公开问卷在手机上易于填写。
- 曲线页面在拿到 M1 面板照片与 CSV 后再做最终定稿。

## 3. 视觉落地规范

采用“现代实验室仪表盘”风格：扁平、简约、明亮、紧凑、数据优先。视觉设计应服务长期使用和快速扫描，避免传统咖啡主题中大面积棕色、复杂阴影和装饰性元素。

### 设计变量

```text
页面背景：#F5F7F8
内容区域：#FFFFFF
主文字：#202628
次要文字：#687477
边框：#DDE3E4
主操作色：#167D6A
强调色：#E06B45
成功状态：#2F8F62
警告状态：#D99524
错误状态：#C94B4B
```

曲线颜色：

```text
豆温：#D6543D
环境温：#D99A2B
ROR：#2677B8
火力：#A34FA3
风门：#278B72
搅拌：#687477
```

字体建议：

```text
中文：Noto Sans SC
英文与数字：IBM Plex Sans
```

组件要求：

- 使用 `4px` 至 `6px` 小圆角。
- 使用细边框和背景层级，不使用明显阴影。
- 避免页面区块和卡片多层嵌套。
- 表格与筛选栏保持紧凑、清晰和易扫描。
- 状态使用小色点或简洁标签。
- 图表区域优先获得页面空间。
- 数字使用等宽数字特性并保持对齐。
- 图标按钮需要提供悬停提示。

### 视觉验收

- 页面整体不能呈现大面积棕色或单一色系。
- 表格、指标和曲线在普通笔记本屏幕下清晰可读。
- 页面层级不依赖阴影表达。
- 同一批次在曲线、指标和评价区域保持一致的颜色标识。
- 温度、时长、重量和百分比数字能够快速横向比较。

## 4. 推荐技术栈

```text
Vue 3
TypeScript
Vue Router
Pinia
ECharts
Vite
```

组件库可以选择成熟的 Vue 管理端组件库，但应统一调整视觉风格，避免默认模板感。

## 5. 项目基础搭建

### 任务

- 初始化 Vue 3 + TypeScript 项目。
- 配置路由、状态管理和基础样式变量。
- 将视觉规范中的颜色、圆角、边框、字体和间距定义为全局设计变量。
- 建立应用壳层：左侧导航、顶部栏、主内容区。
- 建立 Mock 数据和 Mock 请求层。
- 建立通用页面状态组件：加载、空数据、错误、无权限。
- 建立基础表格、筛选栏、抽屉、弹窗和评分组件。

### 验收

- 所有主导航入口可访问。
- 页面刷新后路由状态正确。
- Mock 请求支持模拟成功、空数据和失败。
- 应用壳层和基础组件符合现代实验室仪表盘视觉规范。

## 6. 阶段一：系统总览

### 任务

- 实现年份切换。
- 实现年度核心指标。
- 实现按月烘焙频率图表。
- 实现豆种分布和产区分布。
- 实现待烘豆列表及完成勾选。
- 实现最近烘豆列表。
- 点击图表或最近批次后跳转到带筛选条件的烘焙分析页。

### Mock 数据

- 年度完成锅数
- 已烘焙豆款数
- 总投豆重量
- 平均失重率
- 月度烘焙数据
- 豆种和产区分布
- 待烘与最近烘焙列表

### 验收

- 切换年份后所有模块同步刷新。
- 勾选待烘计划完成后，完成锅数和列表状态同步变化。
- 最近烘焙记录可进入对应曲线页面。

## 7. 阶段二：生豆管理

### 任务

- 实现搜索栏、筛选栏和录入按钮同栏布局。
- 实现生豆三级树状表格。
- 实现录入生豆抽屉。
- 输入生豆名称时搜索已有档案。
- 已存在时仅录入采购批次。
- 不存在时录入生豆档案和首个采购批次。
- 实现库存调整记录界面。
- 点击第三级烘焙记录进入曲线分析页。

### 三级结构

```text
生豆档案 / 豆款
└─ 采购批次
   └─ 烘焙批次
```

### 验收

- 搜索与筛选能够正确影响第一级豆款。
- 展开后可追溯每次采购和每次烘焙消耗。
- 剩余库存只读展示，不能直接编辑。
- 新增采购后列表和库存正确刷新。

## 8. 阶段三：烘焙批次列表

### 任务

- 实现批次搜索、筛选、排序和分页。
- 实现新建待烘计划。
- 实现标记完成与取消完成。
- 实现 M1 CSV 上传入口和解析状态展示。
- 实现曲线状态、评价状态和复盘状态。
- 支持勾选 2 至 4 个已有曲线的批次。
- 支持进入单锅分析或多锅对比。

### 验收

- 无曲线批次不可加入曲线对比。
- 标记完成后批次状态和库存展示同步变化。
- 筛选条件可保留在 URL 查询参数中。

## 9. 阶段四：曲线分析与多锅对比

### 前置材料

- 狙击手 M1 面板照片
- 狙击手 M1 示例 CSV
- CSV 字段说明

### 任务

- 按 M1 习惯组织曲线分析页。
- 默认显示豆温、环境温、ROR、火力、风门、搅拌和关键事件。
- 实现图例点击隐藏与恢复。
- 实现悬停查看同一时间点全部指标。
- 实现时间轴缩放。
- 实现单锅和多锅模式切换。
- 实现批次颜色统一映射。
- 实现关键指标和评价摘要展示。
- 实现进入批次复盘的入口。

### 验收

- 首次加载默认显示全部指标。
- 指标量纲不同但不会造成明显误读。
- 多锅对比时可按批次和按指标控制显隐。
- 同一批次在曲线、指标和评价区域使用相同颜色。
- 桌面宽屏和普通笔记本视口均无重叠。

## 10. 阶段五：公开评价问卷与评价管理

### 任务

- 实现从烘焙批次发起问卷的交互。
- 实现公开问卷移动端页面。
- 实现五分制评分、风味描述和自由备注。
- 实现问卷汇总列表。
- 实现问卷结果详情。
- 实现评分平均值、分布和评价明细。
- 实现复制链接、二维码展示、关闭和延期。
- 实现返回关联烘焙批次及进入综合复盘。

### 验收

- 公开问卷无需登录即可填写。
- 问卷不展示曲线、豆商描述或他人评价。
- 问卷提交后自动关联正确的烘焙批次。
- 评价管理页面不能创建问卷。

## 11. 阶段六：批次复盘

### 任务

- 实现个人初始复盘。
- 实现系统生成的评价反馈摘要。
- 实现综合复盘。
- 实现下一锅调整建议。
- 实现下一次烘焙提醒 1、2、3。
- 实现从建议创建下一次待烘计划。
- 保留个人初始复盘，不被综合复盘覆盖。

### 验收

- 复盘可在没有评价时创建。
- 收到评价后可以继续更新综合复盘。
- 下一次提醒最多三条，并按顺序展示。
- 创建下一次计划时可带入提醒和烘焙目标。

## 12. 阶段七：系统设置

### 任务

- 实现标准词表管理。
- 实现烘焙机与 CSV 配置占位页。
- 实现用户管理占位页。
- 实现文件与备份状态页。

### 验收

- 已使用的词表值只能停用，不能直接删除。
- 设置项变化能够反映到相应录入控件中。

## 13. 前后端协作契约

每完成一个页面，记录以下内容：

- 页面读取的数据
- 筛选、排序和分页参数
- 用户可执行的操作
- 操作后的刷新范围
- 数据加载、空状态和错误状态
- Mock 请求与响应示例

建议维护：

```text
docs/data-contract.md
docs/api-contract.md
docs/frontend-pages.md
frontend/mock/
```

接口设计原则：

- 核心接口围绕数据对象和业务动作。
- 聚合接口围绕完整页面和分析场景。
- 前端不直接承担库存、失重率、养豆天数等业务计算。

## 14. 推荐开发顺序

```text
1. 应用壳层与 Mock 基础
2. 生豆管理
3. 烘焙批次列表
4. 系统总览
5. 曲线分析与多锅对比
6. 公开评价问卷与评价管理
7. 批次复盘
8. 系统设置
9. 接入真实后端接口
10. 响应式、可用性与上线检查
```

生豆管理和批次列表先做，因为它们定义核心录入与筛选模式；曲线页等待 M1 材料后定稿。

## 15. 第一版完成标准

- 用户可以录入生豆和采购批次。
- 用户可以查看采购、库存和烘焙消耗的三级关系。
- 用户可以创建待烘计划并标记完成。
- 用户可以筛选烘焙批次并查看单锅曲线。
- 用户可以选择 2 至 4 锅进行曲线对比。
- 用户可以从批次发起公开评价问卷。
- 用户可以查看评价汇总并完成批次复盘。
- 用户可以记录下一锅建议和三条烘焙提醒。
- 公开问卷在手机端可正常使用。
- 所有关键页面具备加载、空数据和错误状态。
- 所有页面符合扁平、简约、数据优先的现代实验室仪表盘视觉规范。
