# 咖啡烘焙数据分析系统：前端设计 Summary

## 1. 产品目标

本系统面向咖啡烘焙师，核心价值是记录生豆与采购信息，并通过狙击手 M1 烘焙曲线、杯测评价和批次复盘，持续分析不同烘焙操作对风味的影响。

系统包含两条核心数据流：

```text
生豆数据采集流：
生豆档案 -> 采购批次 -> 库存与原料状态

烘焙曲线分析流：
采购批次 -> 烘焙批次 -> M1 CSV 解析 -> 曲线展示与对比
-> 个人初始复盘 -> 问卷评价 -> 综合复盘 -> 下一锅建议
```

其中，烘焙曲线分析是系统的核心功能。

## 2. 前端设计原则

- 前端首先服务日常工作流，而不是简单展示数据库表。
- 高频操作应在两次点击内到达。
- 生豆录入与生豆查看职责分开，但放在同一个工作页面中。
- 问卷必须从具体烘焙批次发起，评价管理只负责汇总与分析。
- 曲线分析默认显示全部关键指标，用户可以主动隐藏。
- 烘焙完成后不强制立即杯测或复盘，允许后续补录。
- 状态用于提醒，不应阻塞用户操作。
- 第一版使用 Mock 数据完成可操作原型，再根据页面需求反推接口。

## 3. 视觉风格

采用“现代实验室仪表盘”风格，整体保持扁平、简约、克制和数据优先，同时通过少量暖色体现咖啡烘焙的温度感。

设计关键词：

```text
明亮
扁平
紧凑
低装饰
专业仪表
清晰数据
安静工作区
```

避免使用大面积咖啡棕色、复杂阴影、装饰性渐变和过度卡片化设计，保证曲线、指标和状态容易区分。

### 3.2 组件规范

- 卡片圆角控制在 `4px` 至 `6px`。
- 不使用明显阴影，主要依靠细边框、留白和背景区分层级。
- 页面区块不嵌套多层卡片。
- 表格保持紧凑但可扫描，适合反复查看和比较。
- 按钮优先使用图标或图标加简短文字。
- 筛选条件集中在紧凑工具栏中。
- 状态使用小色点或简洁标签。
- 曲线图使用尽可能宽的展示区域。
- 左侧导航固定，当前模块使用浅色背景和主操作色标识。
- 温度、时长、重量、百分比等数字保持对齐，便于比较。

### 3.3 字体建议

```text
中文：Noto Sans SC
英文与数字：IBM Plex Sans
```

数字应启用等宽数字特性，提升指标和表格的可读性。


顶部区域提供：

公开评价问卷不出现在后台主导航中，评价者通过公开链接或二维码访问。

## 5. 已确定的数据对象

```text
生豆档案 1 -> N 采购批次
采购批次 1 -> N 烘焙批次
烘焙批次 1 -> 1 当前生效烘焙曲线
烘焙批次 1 -> N 评价问卷
评价问卷 1 -> N 杯测评价记录
烘焙批次 1 -> 1 批次复盘
```

辅助对象包括：

- 库存调整记录
- 标准词表
- 用户
- 评价者
- 文件附件

## 7. 系统总览

系统总览默认展示当前年度，并支持切换年份。

年度聚合数据：

- 已完成烘焙锅数
- 已烘焙豆款数
- 总投豆重量
- 平均失重率
- 按月烘焙频率
- 烘焙豆种分布
- 烘焙产区分布

待烘豆列表：

- 计划烘焙时间
- 生豆名称
- 采购批次
- 计划投豆重量
- 烘焙目标
- 是否完成

勾选完成后，烘焙批次状态变为已完成，年度完成锅数通过实时统计自动增加。

最近烘豆列表展示近期已完成批次，点击后进入曲线分析页面。

## 8. 生豆管理

页面顶部将搜索栏、筛选条件和“录入生豆”按钮组织在同一区域，下方使用三级树状表格统一查看数据。

```text
生豆档案 / 豆款
└─ 采购批次：采购总量、含水率、调整量、剩余库存
   └─ 烘焙批次：烘焙时间、投豆量、出豆量、失重率、状态
```

录入逻辑：

```text
输入或搜索生豆名称
-> 已有生豆：只新增采购批次
-> 没有生豆：创建生豆档案并新增首个采购批次
```

库存实时计算：

```text
剩余库存
= 采购总量
- 已完成烘焙批次投豆量合计
+ 库存调整量合计
```

库存不能直接修改，实际损耗或盘点误差通过库存调整记录处理。

## 9. 烘焙分析

### 9.1 烘焙批次列表

列表用于筛选批次、查看状态、进入单锅分析及选择多锅对比。

主要筛选条件：

- 时间范围
- 生豆名称
- 豆种
- 处理法
- 产区
- 采购批次
- 批次状态
- 是否已有曲线
- 是否已有评价

主要展示字段：

- 烘焙时间
- 生豆名称、豆种、处理法、产区
- 采购批次
- 投豆量、出豆量
- 总时长、发展率、失重率
- 曲线状态、评价数量、批次状态

操作：

```text
点击单条批次 -> 单锅曲线分析
勾选 2 至 4 个批次 -> 多锅曲线对比
```

### 9.2 曲线分析页面

单锅与多锅分析共用一个页面。

首次加载默认展示：

- 豆温
- 环境温
- ROR
- 火力
- 风门
- 搅拌
- 回温点
- 转换点
- 一爆开始
- 下豆点

用户可通过图例逐项隐藏或恢复指标。所有指标共用时间轴，但使用适合自身量纲的纵轴。火力、风门和搅拌使用阶梯线。

多锅模式要求：

- 同时对比 2 至 4 锅
- 每个批次使用固定颜色
- 支持按批次隐藏
- 支持按指标隐藏
- 曲线颜色与指标、评价颜色保持一致

曲线页面最终组织方式需要参考狙击手 M1 面板照片和示例 CSV 后确认。

## 10. 评价管理

问卷只能从烘焙批次发起；评价管理不创建问卷，只负责查看、跟踪和分析。

公开问卷评分维度采用五分制：

- 干香
- 湿香
- 酸感
- 甜感
- 苦感
- 回味
- 综合喜好

其中酸感和苦感表示强度，其他维度表示质量或喜好程度。

评价结果页展示：

- 问卷状态、链接、二维码、截止时间
- 提交人数
- 各维度平均分和分布
- 高频风味描述
- 评价明细
- 不同评价者类型与养豆天数下的差异

## 11. 批次复盘

复盘是对前面事实数据的评论与总结，不重复保存曲线和评价原始数据。

页面区域：

```text
批次概览
个人初始复盘
评价反馈摘要
综合复盘
下一锅调整建议
下一次烘焙提醒 1、2、3
```

个人初始复盘在外部反馈前填写并保留；综合复盘结合曲线和评价后更新。

下一次烘焙提醒最多三条，要求简短、明确、可执行，并可在创建下一次待烘计划时带入。
