## 今日完成工作

### 1. 前端修改（v0.3.0）

根据《咖啡烘焙系统_前端修改指南_GitHub审查版》对前端进行全面修改：

**P0 — 演示环境与规则修正：**
- AppLayout 顶部增加"演示环境"橙色标签，侧栏底部改为"演示环境"
- 修正分享链接格式，支持 GitHub Pages 哈希路由（`/#/eval/{code}`）
- 统一事件名：转换点 → 转黄点（turning → yellowing）
- RoR 单位从 °C/30s 改为 °C/分钟
- 曲线数据模型对齐 Kaleido M1 真实 CSV 字段（beanTempCelsius、rorCelsiusPerMinute、heatingPowerPercent 等）
- Dashboard 月度烘焙频率改为真实聚合（不再用随机数）

**P1 — 操作闭环：**
- 烘焙列表增加业务状态列（待烘/已完成/已作废）
- 增加资料完整度列（缺出豆量/缺CSV/缺评价/缺复盘）
- 增加批次操作弹窗（标记完成、补录出豆量、取消完成、作废）
- 问卷按钮仅在已完成批次显示
- 创建待烘计划增加库存校验（投豆量>0、不超过库存、日期不为空）
- 切换生豆自动重置采购批次选择

**P2 — 复盘闭环：**
- 创建下一锅时弹出确认弹窗
- 提醒携带快照（sourceReviewId、sourceRoastingBatchId、targetRoastingBatchId）
- 评价提交时自动计算养豆天数

**P3 — 系统设置与体验：**
- 烘焙机/CSV 配置页替换为 M1 真实规则
- 文件备份页显示为"未配置"

### 2. 后端全部代码（Phase 1-8）

基于《咖啡烘焙系统_后端设计指南_适配新版前端》实现 FastAPI 后端。

**项目结构：`backend/`，52 个文件，~4642 行代码**

| 层级 | 文件 | 说明 |
|------|------|------|
| 入口 | `app/main.py` | FastAPI app，CORS，异常处理 |
| 核心 | `app/core/` | config、database(asyncio)、security(JWT)、exceptions(16种业务异常)、logging |
| 模型 | `app/models/all_models.py` | 15 张 SQLAlchemy 2.0 async 模型 |
| 模式 | `app/schemas/all_schemas.py` | 全部 Pydantic 请求/响应 schema |
| API | `app/api/v1/` | 10 个路由模块（auth、terms、green_beans、purchase_batches、roasting_batches、curves、questionnaires、evaluations、reviews、dashboard） |
| 仓库 | `app/repositories/` | 8 个数据访问层 |
| 服务 | `app/services/` | 5 个业务逻辑模块（inventory、roasting、curve、questionnaire、review） |
| 解析器 | `app/parsers/kaleido_m1.py` | KLDO V101 CSV 解析器（阶段指标、AUC、控制变化、数据质量检查） |
| 部署 | `Dockerfile`, `docker-compose.yml` | PostgreSQL 17 + FastAPI |

**15 张数据表：**
users, standard_terms, green_beans, purchase_batches, inventory_adjustments, roasting_batches, curve_files, roasting_curves, questionnaires, cupping_evaluations, batch_reviews, review_reminders, batch_reminders, inventory_ledger
