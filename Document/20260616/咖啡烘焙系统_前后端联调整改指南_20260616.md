## 4. 当前首要失败点：生豆录入

用户操作“录入生豆”失败，排查结论：

### 4.1 前端问题

文件：

```text
frontend/src/views/GreenBeanManagement.vue
```

当前仍直接使用 Mock：

```ts
import {
  mockGreenBeans, mockPurchaseBatches, mockRoastingBatches,
  apiCreateGreenBean, apiCreatePurchaseBatch, apiCreateRoastingBatch,
} from '../mock'
```

因此真实模式下：

- 生豆列表不是来自 PostgreSQL。
- 新增生豆不会写入后端。
- 刷新页面后新增数据会丢失。
- 页面错误无法反映后端真实业务错误。

### 4.2 后端问题

接口：

```text
POST /api/v1/green-beans/with-first-purchase
```

直接调用真实后端会返回 500。

根因：

```text
sqlalchemy.exc.MissingGreenlet
```

触发点：

```python
return {
    "green_bean": _gb_to_response(gb),
    ...
}
```

`_gb_to_response(gb)` 内部访问：

```python
gb.variety.value
gb.process.value
```

但 `gb.variety` 和 `gb.process` 是 SQLAlchemy relationship。Async SQLAlchemy 不能在普通属性访问时隐式 lazy load，否则会触发 `MissingGreenlet`。

另外，当前后端还存在两个生豆模块契约缺口：

- `supplier` 入参没有写入 `PurchaseBatch.supplier_term_id`。
- `GET /api/v1/green-beans/tree` 只返回生豆列表，没有返回前端树表需要的采购批次和烘焙批次。

### 5.2 修复 MissingGreenlet

禁止在刚创建的 `gb` 上访问懒加载关系：

```python
gb.variety
gb.process
```

推荐做法一：使用已解析出来的 term 直接构造响应。

示意：

```python
variety_term = None
process_term = None

if body.variety:
    variety_term = await term_repo.get_or_create_value("variety", body.variety)
    variety_term_id = variety_term.id

if body.process:
    process_term = await term_repo.get_or_create_value("process", body.process)
    process_term_id = process_term.id
```

返回时不要调用 `_gb_to_response(gb)`，而是显式传值：

```python
return {
    "green_bean": {
        "id": gb.id,
        "name": gb.name,
        "variety": variety_term.value if variety_term else None,
        "process": process_term.value if process_term else None,
        "region": gb.region,
        "country": gb.country,
        "farm": gb.farm,
        "elevation": gb.elevation,
        "brand": gb.brand,
        "harvest_season": gb.harvest_season,
        "vendor_flavor_description": gb.vendor_flavor_description,
        "first_created_at": gb.first_created_at.isoformat() if gb.first_created_at else None,
    },
    ...
}
```

推荐做法二：创建后重新 eager load：

```python
result = await db.execute(
    select(GreenBean)
    .options(
        selectinload(GreenBean.variety),
        selectinload(GreenBean.process),
    )
    .where(GreenBean.id == gb.id)
)
gb = result.scalar_one()
```

然后再调用 `_gb_to_response(gb)`。

二选一即可。更推荐做法一，修改面小。

```ts
import { apiRequest } from './http'
import type {
  GreenBeanMatchDto,
  GreenBeanTreeDto,
  GreenBeanWithFirstPurchaseRequestDto,
  PurchaseBatchCreateRequestDto,
} from './types'

export function getGreenBeanTree(params: {
  search?: string
  variety?: string
  process?: string
  region?: string
} = {}) {
  const qs = new URLSearchParams()
  if (params.search) qs.set('search', params.search)
  if (params.variety) qs.set('variety', params.variety)
  if (params.process) qs.set('process', params.process)
  if (params.region) qs.set('region', params.region)

  const query = qs.toString()
  return apiRequest<GreenBeanTreeDto[]>(`/green-beans/tree${query ? `?${query}` : ''}`, {
    auth: true,
  })
}

export function matchGreenBeans(name: string) {
  const qs = new URLSearchParams({ name })
  return apiRequest<GreenBeanMatchDto[]>(`/green-beans/matches?${qs}`, {
    auth: true,
  })
}

export function createGreenBeanWithFirstPurchase(body: GreenBeanWithFirstPurchaseRequestDto) {
  return apiRequest('/green-beans/with-first-purchase', {
    method: 'POST',
    body,
    auth: true,
  })
}

export function addPurchaseBatch(greenBeanId: string, body: PurchaseBatchCreateRequestDto) {
  return apiRequest(`/green-beans/${greenBeanId}/purchase-batches`, {
    method: 'POST',
    body,
    auth: true,
  })
}
```

## 12. 必须统一的错误处理

前端 `apiRequest` 已经初步处理：

- 401 清除 token。
- JSON 错误读取 `message`。
- 网络错误显示“网络连接失败”。

后续页面中不要写：

```ts
catch {
  alert('保存失败')
}
```

应该写：

```ts
catch (err) {
  alert(err instanceof ApiError ? err.message : '保存失败')
}
```

```text
loading
errorMessage
retry
```
